# Office Memo (Predictive)

A predictive interoffice memorandum: an **objective** analysis that tells the reader what the law is and how the question will likely come out, weaknesses and all. Draft it from the proof skeleton produced by `dingduff-legal-analysis`.

## A memo predicts; it does not advocate

- **Be objective.** Predict the likely outcome and give honest confidence. Present both sides; don't slant law or facts toward a desired result.
- **Confront weaknesses candidly** — the unfavorable case, the bad fact, the plausible counterargument. A memo that hides the downside sets the reader up to be ambushed.
- **Skeleton before prose.** Build and verify the analytical skeleton before drafting the Discussion.

## Anatomy (defer to firm house style)

1. **Heading** — To / From / Date / Re (a specific Re line: client/matter + the question).
2. **Question(s) Presented** — one per issue, numbered (under-does-when). See `components.md`.
3. **Brief Answer(s)** — bottom line first (Yes / No / Probably…), then the determinative reason. Reader should get the result from QP + Brief Answer alone. Must match the Discussion.
4. **Statement of Facts** — complete, neutral, record-cited. See `components.md`.
5. **Discussion** — the heart. For each issue, a conclusion-first proof (CRAC/CREAC) from the skeleton: state the conclusion, prove the rule, apply it to the facts, **assess** counterarguments (not just rebut), conclude. Objective; depth calibrated to difficulty; multi-issue memos use headings and a roadmap.
6. **Conclusion** — synthesize the predicted outcomes; note key risks, assumptions, open questions, next steps.

## Predictive discussion vs. a brief

States conclusions as **predictions** ("a court would likely hold…"); gives **both sides** fair treatment; **assesses** counterarguments (sometimes the honest answer is "this could go either way"); no theme, no emphasis games, no persuasion pass.

## Template

```
MEMORANDUM
TO / FROM / DATE / RE: [matter] — [the question]

QUESTION(S) PRESENTED
  1. [under-does-when]

BRIEF ANSWER(S)
  1. [Yes/No/Probably]. [Rule + determinative reason.]

STATEMENT OF FACTS
  [Complete, neutral, record-cited.]

DISCUSSION
  [Roadmap if multi-issue.]
  I. [Issue] — [CRAC: conclusion → rule → proof → application → counterargument assessment → conclusion]
     A. [sub-issue/element]   (nest as needed)

CONCLUSION
  [Predicted outcomes; risks; assumptions; next steps.]
```

Match length to difficulty: most words go to the contested issues; an undisputed element gets a sentence.

## Self-check

Objective throughout; every issue addressed and weaknesses confronted; skeleton-anchored before prose; QP ↔ Brief Answer ↔ Discussion ↔ Conclusion consistent; facts complete and neutral; clarity pass run; citations checked.

## Going further (opinion-grade memos)

For bet-the-matter questions, formal opinions, or novel issues: do the exhaustive version, following the "Going further" guidance in `dingduff-legal-analysis` (exhaustive issue inventory, every rule formulation, alternative proof structures). Add an **alternative-outcomes** treatment (lay out the alternatives, conditions, and probabilities where an issue is genuinely uncertain), present the **strongest contrary analysis** for each dispositive issue, and add an explicit **risk/assumptions** section (assumptions the prediction rests on and how it changes if each fails; gap facts; unsettled law; per-issue confidence). Stay objective despite the depth.
